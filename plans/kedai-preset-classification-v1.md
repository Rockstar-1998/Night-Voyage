---

## 当前分类原则

本表遵循以下原则：

1. 高频、稳定、强互斥、用户容易理解的内容 -> 语义组
2. 个性化、补丁式、作者强风格内容 -> 自由 block
3. 真正的示例对话 -> `preset_examples`
4. 采样与生成控制 -> 参数层 / prefill
5. 不应再由预设承担的功能 -> 延后迁出或挂到未来系统

## few-shot 与 examples 的初步判断

在本轮已读取的代表性条目中，绝大多数是：

- 规则说明
- 文风定义
- 输出模板
- 特殊功能约束

而不是标准的 `user -> assistant` few-shot 示例对。

因此当前第一版判断是：

- 可待迁移的第一阶段应优先完成 `语义组 + 自由 block + params`
- `preset_examples` 只在后续发现明确的示例对话材料后再单独抽出

也就是说：

**第一版 Night Voyage 可待预设，examples 可以先留空或仅保留极少量明确示例。**

---

## 参数层的初步判断

可待原始预设中确实有采样参数与 prefill 相关能力，但第一版迁移建议：

- 只迁移明确、稳定、已被 Night Voyage 支持的参数
- 暂不把 ST 的前后夹 prompt 技巧误当作参数功能

当前建议优先迁移：

- `temperature`
- `max_output_tokens`
- `top_p`
- 已支持的 penalties / response mode（如确有对应诉求）
